⚙️ CORE SYSTEM FILES (Production Code)
──────────────────────────────────────

[7] rag_conflict_resolver.py (16 KB - MAIN ENGINE)
    Purpose: Core conflict detection and resolution system
    Classes:
    ├─ Document: Loaded web page with url, title, content
    ├─ Entity: Extracted entity with multiple versions
    ├─ Conflict: Conflicting information (4 severity levels)
    ├─ ConflictDetector: Main analysis engine
    │  ├─ extract_entities(): Regex-based entity extraction
    │  ├─ add_document(): Process docs and detect conflicts
    │  ├─ _create_conflict(): Severity scoring
    │  └─ get_conflicts(): Query interface
    ├─ InteractiveResolver: User decision interface
    │  ├─ display_conflict(): Show versions to user
    │  ├─ get_user_choice(): Get user input
    │  └─ resolve_interactive(): Full resolution workflow
    └─ KnowledgeBaseUpdater: Apply resolutions to KB
       ├─ update_from_resolutions(): Batch update
       └─ export_kb(): Export cleaned KB
    
    Key Features:
    ✓ Entity types: person, location, organization, date, number
    ✓ Severity levels: critical, high, medium, low
    ✓ Confidence scoring: 0-100%
    ✓ Source tracking: URL + timestamp
    ✓ Resolution history: Full audit trail
    ✓ KB persistence: JSON export/import
    
    Use This When: You need to detect and resolve conflicts
    Dependencies: json, dataclasses, datetime, regex
    Examples: See rag_cleanup_interactive_demo.py


[8] rag_progressive_stress_tests.py (20 KB - TEST FRAMEWORK)
    Purpose: 4-level progressive difficulty testing with crash detection
    Test Levels:
    ├─ Level 1 EASY: 50 docs, 5 simple conflicts → ✅ PASS
    ├─ Level 2 MEDIUM: 100 docs, 20 complex conflicts → ✅ PASS
    ├─ Level 3 HARD: 200 docs, type mismatches → ❌ CRASH → ✅ FIX
    └─ Level 4 EXTREME: 500 docs, circular refs → ✅ PASS
    
    Key Features:
    ✓ Automatic test data generation
    ✓ Crash detection and recording
    ✓ Automatic fix application
    ✓ Performance measurement
    ✓ Detailed reporting
    
    Run With: python rag_progressive_stress_tests.py
    Output: Test results printed to console + summary JSON
    Success Criteria: All 4 levels pass
