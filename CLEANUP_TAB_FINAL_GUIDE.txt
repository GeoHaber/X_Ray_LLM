VISION: INTERACTIVE CLEANUP TAB
================================================================================

Current Problem:
  ✗ Multiple documents with conflicting information
  ✗ City manager known by different names (Mihai Jurca, Michael Jurka, M. Iurca)
  ✗ Population varies across sources (176248, 175500, ~176000)
  ✗ Manual resolution required = slow KB updates

Solution: Interactive Cleanup Tab in RAG_RAT

  1. SCAN 200+ documents automatically
  2. DETECT conflicts in real-time
  3. SHOW user side-by-side comparison
  4. LET user decide which version to keep
  5. UPDATE knowledge base with chosen values
  6. TRACK resolution history for audit trail

================================================================================
HOW IT WORKS
================================================================================

Phase 1: SCAN & DETECT
  Input:  200+ documents from web crawl
  Process: Extract entities, find variations
  Output: Conflict list with confidence scores
  
  Example:
    Entity: Mayor
    Versions:
      [1] Florin Birta (95%) ← 12 sources
      [2] Florin Birtă (87%) ← 8 sources  
      [3] F. Birta (72%)     ← 2 sources

Phase 2: INTERACTIVE RESOLUTION
  Show: Visual comparison of all versions
  Input: User clicks button to choose version
  Logic: Tracks user decision
  Update: Auto-updates knowledge base

Phase 3: AUTO-RESOLUTION
  Smart: Uses confidence > 85% to auto-resolve
  Track: Records all decisions
  Learn: System learns from user patterns

Phase 4: CONTINUOUS IMPROVEMENT
  Periodic: Scan documents again
  Compare: Find new conflicts
  Update: Keep KB synchronized
